Elementor Integration Guidelines

Core Principle: Elementor is used for structure and layout only. All visual styling is controlled by theme CSS.

Elementor's Role:

  • Structure: Page layout, sections, columns, widgets
  • Content: Text, images, media placement
  • Organization: Content hierarchy and flow
  • Responsive: Breakpoint management and visibility

Theme CSS Role:

  • Colors: All background, text, border colors
  • Typography: Fonts, sizes, weights, spacing
  • Spacing: Padding, margins, gaps
  • Effects: Shadows, borders, transitions
  • Layout: Container widths, grid systems
Best Practices:
  • Apply CSS classes in Elementor's "Advanced → CSS Classes" field
  • Avoid using Elementor's Style tab for colors/typography
  • Use Elementor's Style tab only for layout (width, alignment)
  • Reference this style guide for available CSS classes
  • Keep all styling in theme CSS for maintainability
Important: Elementor Site Settings (Theme Style) have been cleared. All styling must come from theme CSS classes.

Usage Guidelines

Elementor Widget Styling:
  • Use Elementor's "Advanced → CSS Classes" to apply theme CSS classes
  • Leave Style tab colors/typography empty - use CSS classes instead
  • Use Style tab only for layout properties (width, alignment, spacing)
  • Reference this guide for available CSS classes and their usage
Maintaining Consistency:
  • Always use design system colors (never create new ones)
  • Follow typography hierarchy (H1-H6 sizes and weights)
  • Use spacing scale values (4px, 8px, 16px, etc.)
  • Apply button styles via CSS classes, not inline styles
  • Keep all custom CSS in theme style.css file

Templates & Theme Structure

Documentation for all page templates, header/footer templates, and Elementor Theme Builder templates used in the EHS site.

Elementor Theme Builder Templates

What are Theme Builder Templates?

Elementor Pro's Theme Builder allows you to create custom templates for headers, footers, single posts, archives, and more. These templates override the default WordPress theme templates.

Header Template

  • Template Type: header
  • Location: Elementor → Theme Builder → Headers
  • Assignment: Applied globally via Theme Builder conditions
  • Fallback: Uses template-parts/header.php if no Elementor header exists
  • Key Features:
    • Logo placement (horizontal logo preferred)
    • Main navigation menu
    • Contact button/CTA
    • Mobile responsive menu
    • Sticky header option
Finding Header Template:
wp post list --post_type=elementor_library --meta_key=_elementor_template_type --meta_value=header

Editing: WordPress Admin → Templates → Theme Builder → Headers

Footer Template

  • Template Type: footer
  • Location: Elementor → Theme Builder → Footers
  • Assignment: Applied globally via Theme Builder conditions
  • Fallback: Uses template-parts/footer.php if no Elementor footer exists
  • Key Features:
    • Company contact information
    • Footer navigation links
    • Social media links
    • Copyright information
    • Uses .ehs-footer CSS classes from theme
Finding Footer Template:
wp post list --post_type=elementor_library --meta_key=_elementor_template_type --meta_value=footer

Editing: WordPress Admin → Templates → Theme Builder → Footers

Single Post Template

  • Template Type: single
  • Location: Elementor → Theme Builder → Single
  • Assignment: Can be assigned to specific post types or all posts
  • Usage: Custom layout for blog posts, articles, etc.

Archive Template

  • Template Type: archive
  • Location: Elementor → Theme Builder → Archive
  • Assignment: Can be assigned to specific post types or taxonomies
  • Usage: Custom layout for post archives, category pages, etc.

Section Templates

  • Template Type: section
  • Location: Elementor → Saved Templates
  • Usage: Reusable content blocks (hero sections, CTAs, service blocks, etc.)
  • Examples:
    • Service cards sections
    • Certification badges
    • Contact forms
    • Testimonial sections

Template Management

List All Elementor Templates:
wp post list --post_type=elementor_library --format=table

Find Specific Template Type:
wp post list --post_type=elementor_library --meta_key=_elementor_template_type --meta_value=header
wp post list --post_type=elementor_library --meta_key=_elementor_template_type --meta_value=footer

Export Template:
wp post get [ID] --format=json > template-backup.json

Sync Templates from Production:
./sync-elementor-templates.sh

View Template in Admin:
WordPress Admin → Templates → Saved Templates (or Theme Builder)
Template Best Practices:
  • Always use CSS classes from theme style.css (never inline styles in Elementor)
  • Keep header/footer templates consistent across all pages
  • Use section templates for reusable content blocks
  • Test templates on mobile devices before publishing
  • Document custom templates in this style guide
  • Export templates before making major changes
